Toolkit — Windows Floating System Monitor

Toolkit is a lightweight, high-quality system status monitor for Windows. Built with Tauri (Rust) and a React + TypeScript frontend, Toolkit provides a borderless, always-on-top floating widget that displays essential system metrics — CPU, memory, network throughput, and battery — with a modern, Material-inspired aesthetic.

Repository: https://github.com/AndrewWangDev/monitor_tool

Key Features

  • Real-time system metrics: CPU usage percentage, memory usage (percentage and absolute), upload/download network throughput, and battery level/charging state.
  • Floating, draggable widget: borderless, always-on-top window that can be positioned anywhere on the screen and dragged via an explicit drag region.
  • Lightweight runtime: Tauri + Rust backend keeps resource usage low compared with typical Electron-based tools.
  • Modern UI: Material-like tokens and dynamic transparency for a clean, unobtrusive look.
  • User-configurable display: selectively show/hide CPU, RAM, Network, or Battery panels via an in-widget settings overlay.

How It Works (Implementation Notes)

  • Frontend: React (v19) + TypeScript, bundled with Vite and styled with TailwindCSS. The UI exposes a drag handle, a compact settings overlay, and metric cards that update every second.
  • Backend: Tauri (Rust) exposes a get_metrics command that returns a JSON payload containing cpu_usage, total_memory, used_memory, total_rx, and total_tx. The frontend calls this command once per second to update the display and compute network speeds as byte differences between samples.
  • System data: Rust uses crates such as sysinfo to read low-level system metrics on Windows.
  • Battery monitoring: the frontend queries the browser navigator.getBattery() when available to show battery level and charging state.

  • Rust toolchain (stable)
  • WebView2 (usually preinstalled on Windows 10/11)

Files and Dependencies of Note

  • package.json: React + Vite + Tauri frontend; key packages include @tauri-apps/api, @tauri-apps/cli, react, vite, and tailwind-merge.
  • src/App.tsx: Main UI component — uses @tauri-apps/api/window for dragging, invokes get_metrics for system data, and computes network speed deltas client-side.
  • src-tauri/Cargo.toml: Rust dependencies include tauri, tauri-plugin-opener, serde, serde_json, and sysinfo for system metrics.

Use Cases

  • Developers, designers, and power users who need a compact overview of system performance without desktop clutter.
  • Users who prefer aesthetic, always-on-top widgets with minimal resource overhead.
